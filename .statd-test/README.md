## 3 things worth keeping in mind:
* Please carefully review the contents of any instance's `export-my-stats.sh` when writing your own for the first time (or after a long break). Notably, the trailing newline after all commands is (as much as I hate trailing newlines) CRUCIAL, otherwise `cat` will smash together one instance's script with another's. _**This is important for** both **prod** and testing**.**_
* The test should be launched by `cd`ing into this dir and then doing `./../statd.sh`. This also means that it'll fail after 1st iteration because `./statd.sh` doesn't exist at that location. This is fine, tho, tbh. It's a basic sanity-check, anyway. _This is relevant for testing._
* Please carefully review the contents of any instance's `export-my-stats.sh` when writing your own for the first time (or after a long break). Notably, the exact flags (`-v` is for logging clarity, **`-u` is needed to avoid unnecessary writes** (only updates older files), and `-r` is `--recursive`) and the copy destination (ie. how all instances share a common export „sink”, so you need to be careful when exporting, to not write over anything pre-existing on accident). _This is relevant for prod._
